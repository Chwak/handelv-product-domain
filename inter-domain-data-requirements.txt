# Product Domain - Inter-Domain Data Requirements

## Overview
All data that the Product Domain consumes from other domains via events and provides to other domains.

**Domain:** Product Domain (unified shelf + inventory features)
**API:** Single unified Product AppSync GraphQL API
**Last Updated:** 2026-01-31

---

## 1. EVENTS CONSUMED

### Shelf Feature - Events Consumed

#### 1.1 From Order Domain
**Event**: `OrderCreated`

```typescript
{
  event: "OrderCreated",
  orderId: string,
  productId: string,
  quantity: number,
}
```

**Action**: Decrement `quantityAvailable`. If 0, update status to 'sold_out'

### 1.2 From Order Domain
**Event**: `OrderCanceled`

```typescript
{
  event: "OrderCanceled",
  orderId: string,
  productId: string,
  quantity: number,
}
```

**Action**: Increment `quantityAvailable`

#### 1.3 From Maker Domain
**Event**: `MakerVerified`

```typescript
{
  event: "MakerVerified",
  makerUserId: string,
  verifiedAt: string,
}
```

**Action**: Enable publishing for verified maker

---

### Inventory Feature - Events Consumed

#### 1.4 From Order Domain
**Event**: `OrderCompleted`

```typescript
{
  event: "OrderCompleted",
  orderId: string,
  shelfItemId: string,
  makerUserId: string,
}
```

**Action**: Suggest recording material usage for this order (optional)

#### 1.5 From Payment Domain
**Event**: `MaterialPurchasePaid`

```typescript
{
  event: "MaterialPurchasePaid",
  makerUserId: string,
  materialId: string,
  quantity: number,
  totalCost: number,
  paidAt: string,
}
```

**Action**: Record stock purchase and update material inventory

## Integration Architecture

**Single Unified Product AppSync API:**
- Merged schema with shelf + inventory types
- All shelf and inventory queries/mutations/subscriptions in one API
- SSM Parameter: `/product-domain/appsync/api-url`
- Event-driven communication via EventBridge

**Internal Coordination:**
- Shelf and inventory features coordinate internally for:
  - Basement storage (unpublished products)
  - Product quantity tracking
  - Material usage linked to products

**Cross-Domain Imports:**
- Imports `maker_profiles` table (read-only) via SSM from Maker Domain
- Does NOT directly write to other domain databases
